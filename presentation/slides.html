Gas Savings Mechanism

Each separate transaction has a 21,000 gas base cost. Batching N calls saves (N-1) × 21,000 gas by paying the base cost only once.

Novel Innovation: CompressedBatchExecutor

Our key differentiator — beyond standard batching

1. executeSameTarget()

When all calls go to one contract (common case: marketplace, DEX), store the target address once instead of N times.

Saves 19 bytes × (N-1) calls

2. executeCompressedBatch()

Target deduplication via uint8 index table. Each call references a target by 1-byte index instead of 20-byte address.

Supports up to 256 unique targets

3. executeWithContext() EIP-2771

Appends original msg.sender to calldata. Target contracts can extract real user identity. Solves the identity loss problem.

4. executeBundledBatches() CROSS-USER

Multiple users' batches in ONE transaction. 3 users × N calls = 1 on-chain tx. All users pay ZERO gas.

Context-Preserving Batch Execution

Solving the msg.sender identity loss problem in batch calls

Standard Batching Problem

User → BatchExecutor → SampleDApp Inside SampleDApp: msg.sender = BatchExecutor // User identity is LOST! // Profile stored under wrong address

When BatchExecutor calls the DApp, msg.sender becomes the BatchExecutor's address, not the user's. This is a fundamental problem.

Our Solution: EIP-2771 Context

User → CompressedBatch → SampleDAppMeta CompressedBatch appends sender: calldata = originalData + user.address SampleDAppMeta extracts it: sender = _msgSender() // Correct user identity preserved!

executeWithContext() appends the original sender to calldata. EIP-2771 compatible targets extract the real user.

Meta-Transaction Flow (EIP-712)

How It Works

1. User builds batch of actions in the UI

2. Frontend creates EIP-712 typed data with calls hash, nonce, and deadline

3. User signs with MetaMask (human-readable prompt)

4. Signed payload sent to Relayer via HTTP POST

5. Relayer validates, checks sponsorship, submits on-chain

6. BatchExecutor verifies signature + executes — user pays ZERO gas

EIP-712 Typed Data Structure

domain = { name: 'GasOptimizer', version: '1', chainId: 31337, verifyingContract: batchExecutor } BatchExecution = { from: address, callsHash: bytes32, nonce: uint256, deadline: uint256 }

Security

  • Calls hash prevents tampering
  • Nonce prevents replay attacks
  • Deadline prevents stale submissions
  • Domain separator prevents cross-chain replay
